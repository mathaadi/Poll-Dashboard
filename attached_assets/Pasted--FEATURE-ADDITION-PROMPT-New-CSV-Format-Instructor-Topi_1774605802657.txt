## CONTEXT — TWO CSV FORMATS NOW EXIST

The app currently handles **Format A** (Zoom export with 9-row header preamble).
A new **Format B** is now also supported — it is a flat CSV with column headers on Row 1.

The app must auto-detect which format is uploaded and parse accordingly.
Both formats store data in the same DB tables — no separate storage.

## FORMAT B — EXACT COLUMN STRUCTURE

Format B CSV has column headers on **Row 1** (no preamble rows):

| Col Index | Column Name (as in header) | Description |
|-----------|---------------------------|-------------|
| 0 | `User Name` | Student full name |
| 1 | `Email Address` | Student email |
| 2 | `Submitted` | Submission timestamp |
| 3 | `Collected` | Collection/session timestamp |
| 4 | `Topic/Name` | Topic of the session (e.g. "Introduction to ML", "Decision Trees") |
| 5 | `Meeting/Webinar ID` | Unique meeting ID — used for duplicate detection (same as Format A) |
| 6 | `Session Rating` | Single numeric rating 1–5 (replaces the two separate delivery/content ratings) |
| 7 | `Instructor` | Instructor/trainer name (e.g. "Dr. Sharma", "Prof. Verma") |
| 8 | `How was the session` | Free text feedback — qualitative question (equivalent to Format A's feedback column) |
| 9 | `Any additional feedback` | Additional free text comments (new — store separately) |

### Format B Parsing Rules:
1. Row 1 = header row (contains `User Name`, `Email Address`, etc.)
2. Rows 2+ = data rows — one row per student response
3. No metadata rows — subject/program info comes from the upload form (user fills in manually, same as today)
4. `Meeting/Webinar ID` in column 5 = use for duplicate detection (same UNIQUE constraint)
5. `Session Rating` is a single value (1–5) — store in both `delivery_rating` AND `content_rating` columns (same value) so existing dashboard charts work without changes
6. `Topic/Name` = the specific topic taught in that session — store in new `topic` column
7. `Instructor` = teacher name — store in new `instructor` column
8. `How was the session` = primary feedback text — map to existing `feedback_text` column
9. `Any additional feedback` = secondary feedback — store in new `additional_feedback` column
10. `Submitted` = use as `submission_time`
11. `Collected` = store as session timestamp if `session_date` is not determinable otherwise

### Auto-Detection Logic (add to `csvParser.js`):
```javascript
function detectCSVFormat(fileContent) {
  const firstLine = fileContent.split('\n')[0].toLowerCase();
  
  // Format B has 'user name' or 'instructor' on the very first line
  if (firstLine.includes('instructor') || firstLine.includes('topic/name') || firstLine.includes('session rating')) {
    return 'FORMAT_B';
  }
  
  // Format A has 'overview' on the first line
  if (firstLine.includes('overview')) {
    return 'FORMAT_A';
  }
  
  // Fallback: check if row 1 looks like headers (contains 'user name' within first 3 lines)
  const first3Lines = fileContent.split('\n').slice(0, 3).join('\n').toLowerCase();
  if (first3Lines.includes('user name') && first3Lines.includes('email')) {
    return 'FORMAT_B';
  }
  
  return 'FORMAT_A'; // default to existing format
}
```

### 1. Update `csvParser.js` — Add Format B Parser

Add a new function `parseFormatB(fileContent)` alongside the existing `parseFormatA`:

```javascript
function parseFormatB(fileContent) {
  const lines = fileContent.split(/\r?\n/).filter(l => l.trim());
  
  // Row 0 = headers, find column indices dynamically
  const headers = parseCSVLine(lines[0]).map(h => h.toLowerCase().trim());
  
  const idx = {
    userName:        headers.findIndex(h => h.includes('user name')),
    email:           headers.findIndex(h => h.includes('email')),
    submitted:       headers.findIndex(h => h.includes('submitted')),
    collected:       headers.findIndex(h => h.includes('collected')),
    topic:           headers.findIndex(h => h.includes('topic')),
    meetingId:       headers.findIndex(h => h.includes('meeting')),
    sessionRating:   headers.findIndex(h => h.includes('session rating') || h.includes('rating')),
    instructor:      headers.findIndex(h => h.includes('instructor')),
    feedback:        headers.findIndex(h => h.includes('how was')),
    additionalFeedback: headers.findIndex(h => h.includes('additional')),
  };
  
  const metadata = {
    meetingId: null,
    sessionDate: null,
    subject: null,     // Will be filled from upload form
    sessionType: null, // Will be filled from upload form
    format: 'FORMAT_B',
    instructor: null,  // Will be extracted from most common value in rows
    topic: null,       // Will be extracted from most common value in rows
  };
  
  const responses = [];
  
  for (let i = 1; i < lines.length; i++) {
    const cols = parseCSVLine(lines[i]);
    if (!cols || cols.length < 3) continue;
    
    const meetingId = idx.meetingId >= 0 ? cols[idx.meetingId]?.trim() : null;
    if (!metadata.meetingId && meetingId) metadata.meetingId = meetingId;
    
    const sessionDate = idx.collected >= 0 ? cols[idx.collected]?.trim() : 
                        idx.submitted >= 0 ? cols[idx.submitted]?.trim() : null;
    if (!metadata.sessionDate && sessionDate) metadata.sessionDate = sessionDate;
    
    const ratingRaw = idx.sessionRating >= 0 ? cols[idx.sessionRating]?.trim() : null;
    const rating = ratingRaw ? parseInt(ratingRaw) : null;
    
    const instructor = idx.instructor >= 0 ? cols[idx.instructor]?.trim() : null;
    const topic = idx.topic >= 0 ? cols[idx.topic]?.trim() : null;
    
    responses.push({
      studentName:        idx.userName >= 0 ? cols[idx.userName]?.trim() : null,
      studentEmail:       idx.email >= 0 ? cols[idx.email]?.trim() : null,
      submissionTime:     idx.submitted >= 0 ? cols[idx.submitted]?.trim() : null,
      deliveryRating:     rating,    // Same value for both
      contentRating:      rating,    // Same value for both (single rating format)
      feedbackText:       idx.feedback >= 0 ? cols[idx.feedback]?.trim() : null,
      additionalFeedback: idx.additionalFeedback >= 0 ? cols[idx.additionalFeedback]?.trim() : null,
      instructor:         instructor,
      topic:              topic,
    });
  }
  
  // Extract dominant instructor and topic from responses (most frequently occurring)
  const instructorCounts = {};
  const topicCounts = {};
  responses.forEach(r => {
    if (r.instructor) instructorCounts[r.instructor] = (instructorCounts[r.instructor] || 0) + 1;
    if (r.topic) topicCounts[r.topic] = (topicCounts[r.topic] || 0) + 1;
  });
  metadata.instructor = Object.entries(instructorCounts).sort((a,b) => b[1]-a[1])[0]?.[0] || null;
  metadata.topic = Object.entries(topicCounts).sort((a,b) => b[1]-a[1])[0]?.[0] || null;
  
  return { metadata, responses };
}
```
