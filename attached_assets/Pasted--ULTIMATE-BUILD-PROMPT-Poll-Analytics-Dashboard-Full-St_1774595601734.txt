## рҹ“Ӯ EXACT CSV FORMAT (CRITICAL вҖ” READ CAREFULLY)

The CSV files come from Zoom's poll export. They are NOT standard CSVs. Here is the exact structure:

```
Row 1:  Overview,,,,,,,,,,
Row 2:  Generate Time,Meeting Topic,Meeting/Webinar ID,Actual Start Time,...
Row 3:  03-09-2026 19:01,ATA || Practical Session,89480443928,02-07-2026 17:05,...
Row 4:  (empty or distribution data вҖ” skip)
Row 5:  Launched Polls,,,,...
Row 6:  #,Poll Name,Questions,Responses,...
Row 7:  1,Untitled poll 1,3,161,...
Row 8:  (empty вҖ” skip)
Row 9:  Untitled poll 1,,,,...
Row 10: #,User Name,Email Address,Submitted Date and Time,[Q1 text],[Q2 text],[Q3 text],...  вҶҗ HEADER ROW
Row 11: 1,Mahi Jain,b25bs2142@iitj.ac.in,02-07-2026 18:35,4,4,fine,...  вҶҗ DATA STARTS HERE
Row 12: 2,Om Jee,omjeemishra202222@gmail.com,02-07-2026 18:35,4,4,...
...
```

### How to Parse:
1. Read Row 3, Column B вҶ’ extract `Meeting Topic` (e.g. `ATA || Practical Session`) вҶ’ subject = `ATA`, session_type = `Practical Session`
2. Read Row 3, Column C вҶ’ `Meeting/Webinar ID` вҶ’ use for duplicate detection
3. Read Row 3, Column D вҶ’ `Actual Start Time` вҶ’ session date (format: `MM-DD-YYYY HH:mm`)
4. Skip rows 1вҖ“9
5. Row 10 = column headers (contains `User Name`, `Email Address`, `Submitted Date and Time`, and the two rating questions)
6. Rows 11+ = actual student response data. Columns are:
   - Col A (index 0): Row number вҖ” ignore
   - Col B (index 1): Student name
   - Col C (index 2): Student email
   - Col D (index 3): Submission timestamp
   - Col E (index 4): **Delivery Rating** (1вҖ“5) вҖ” "How would you rate the overall delivery..."
   - Col F (index 5): **Content Rating** (1вҖ“5) вҖ” "How would you rate the quality and relevance..."
   - Col G (index 6): **Feedback Text** (free text, often empty)
7. Stop reading when you hit a row where Col B starts with empty or whitespace only

### Important Notes:
- Always open files with `utf-8-sig` encoding (handles BOM character)
- Ratings can be blank вҖ” store as NULL
- Emails can have typos (e.g. `iit.ac.in` instead of `iitj.ac.in`, `gmai.com` instead of `gmail.com`) вҖ” store as-is, don't reject
- Some student names include roll numbers in parentheses e.g. `Aashman Sharma (B25BS2003)` вҖ” store full string
- The right side of rows 4вҖ“9 contain Zoom-generated distribution summary data (columns IвҖ“K approximately) вҖ” **ignore these, compute your own**

### Cohort Detection from Email:
- Email starts with `b25bs` вҶ’ Cohort = `2025-BS`
- Email starts with `b24bs` вҶ’ Cohort = `2024-BS`
- Email starts with `b23bs` вҶ’ Cohort = `2023-BS`
- Anything else вҶ’ Cohort = `External/Unknown`

## рҹ—ғпёҸ DATABASE SCHEMA (SQLite)

Use **SQLite** with the `better-sqlite3` npm package. Database file at `./db/polls.db`.

### Table 1: `uploads`
```sql
CREATE TABLE IF NOT EXISTS uploads (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  meeting_id TEXT NOT NULL,
  meeting_topic TEXT,
  subject TEXT NOT NULL,
  session_type TEXT NOT NULL,
  program_name TEXT NOT NULL,
  cohort TEXT,
  semester TEXT,
  week_number TEXT,
  session_date TEXT,
  upload_timestamp TEXT DEFAULT (datetime('now')),
  total_responses INTEGER DEFAULT 0,
  avg_delivery_rating REAL,
  avg_content_rating REAL,
  avg_combined_rating REAL,
  feedback_count INTEGER DEFAULT 0,
  nps_delivery REAL,
  nps_content REAL,
  UNIQUE(meeting_id)
);
```

### Table 2: `poll_responses`
```sql
CREATE TABLE IF NOT EXISTS poll_responses (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  upload_id INTEGER REFERENCES uploads(id),
  student_name TEXT,
  student_email TEXT,
  cohort TEXT,
  submission_time TEXT,
  delivery_rating INTEGER,
  content_rating INTEGER,
  feedback_text TEXT,
  feedback_sentiment TEXT,
  feedback_themes TEXT,
  is_useful_feedback INTEGER DEFAULT 0
);
```

### Table 3: `students`
```sql
CREATE TABLE IF NOT EXISTS students (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT UNIQUE,
  name TEXT,
  cohort TEXT,
  first_seen TEXT,
  last_seen TEXT,
  total_sessions INTEGER DEFAULT 0
);
```

## рҹ§  NLP CLASSIFICATION (Pure JavaScript вҖ” No Libraries)

Add this logic in `backend/src/services/nlp.js`. This runs at upload time and stores results in DB.

### Step 1: Filter Useful Feedback
```javascript
function isUsefulFeedback(text) {
  if (!text || text.trim().length === 0) return false;
  const cleaned = text.trim().toLowerCase();
  
  // Too short (3 words or less)
  if (cleaned.split(/\s+/).length <= 3) return false;
  
  // Junk words
  const junkExact = ['good', 'fine', 'ok', 'okay', 'nice', 'great', 'n/a', 'na', 'no', 
    'yes', 'nothing', 'none', 'nope', 'no comments', 'no comment', 'no issues',
    'good session', 'nice session', 'great session', 'very good', 'very nice',
    'all good', 'all fine', 'everything is fine', 'everything was good',
    'no feedback', 'nothing to say', '...', '..', '.', '-', 'nil'];
  if (junkExact.includes(cleaned)) return false;
  
  // Repeated characters (e.g. "......", "!!!")
  if (/^(.)\1{4,}$/.test(cleaned)) return false;
  
  return true;
}
```

### Step 2: Classify Sentiment
```javascript
function classifySentiment(text) {
  const t = text.toLowerCase();
  
  const negativeWords = ['bad', 'poor', 'worst', 'terrible', 'horrible', 'boring', 'waste',
    'confusing', 'confused', 'unclear', 'didn\'t understand', 'hard to follow',
    'too fast', 'very fast', 'breaking', 'broke', 'lag', 'lagging', 'disconnected',
    'camera off', 'no camera', 'camera was off', 'zero interaction', 'no interaction',
    'not clear', 'not good', 'not helpful', 'not engaging', 'slow', 'too slow',
    'couldn\'t hear', 'audio issue', 'voice breaking', 'mic issue', 'network issue'];
  
  const positiveWords = ['excellent', 'amazing', 'wonderful', 'fantastic', 'love', 'loved',
    'great explanation', 'well explained', 'very clear', 'very helpful', 'interesting',
    'enjoyed', 'engaging', 'interactive', 'informative', 'insightful', 'effective',
    'knowledgeable', 'good examples', 'real world', 'practical example', 'understood',
    'easy to understand', 'well structured', 'very good session', 'learned a lot'];
  
  const suggestionWords = ['suggest', 'suggestion', 'should', 'could', 'would be better',
    'please', 'try to', 'recommend', 'improve', 'improvement', 'next time',
    'in future', 'would like', 'can you', 'if possible', 'it would be nice',
    'maybe', 'perhaps', 'consider', 'add more', 'more examples', 'more practice'];
  
  const negCount = negativeWords.filter(w => t.includes(w)).length;
  const posCount = positiveWords.filter(w => t.includes(w)).length;
  const sugCount = suggestionWords.filter(w => t.includes(w)).length;
  
  if (sugCount > 0 && negCount > 0) return 'suggestion';
  if (sugCount > posCount) return 'suggestion';
  if (negCount > posCount) return 'negative';
  if (posCount > 0) return 'positive';
  return 'neutral';
}
```

### Step 3: Extract Themes
```javascript
function extractThemes(text) {
  const t = text.toLowerCase();
  const themes = [];
  
  const themeMap = {
    'pace': ['pace', 'speed', 'fast', 'slow', 'rushed', 'too quick', 'hurried'],
    'audio': ['audio', 'voice', 'mic', 'microphone', 'sound', 'hear', 'hearing', 'breaking', 'echo'],
    'video': ['camera', 'video', 'screen', 'visual', 'display', 'share screen'],
    'content': ['content', 'topic', 'material', 'slides', 'notes', 'examples', 'theory'],
    'delivery': ['delivery', 'explanation', 'explained', 'teaching', 'trainer', 'instructor'],
    'interaction': ['interaction', 'interactive', 'engage', 'engaging', 'participation', 'q&a', 'questions'],
    'language': ['language', 'hindi', 'english', 'terminology', 'terms'],
    'network': ['network', 'connection', 'internet', 'lag', 'buffering', 'disconnect'],
    'tools': ['tool', 'software', 'platform', 'zoom', 'code', 'script', 'demo']
  };
  
  for (const [theme, keywords] of Object.entries(themeMap)) {
    if (keywords.some(kw => t.includes(kw))) {
      themes.push(theme);
    }
  }
  
  return themes; // array of theme strings
}
```

## вҡҷпёҸ BACKEND LOGIC вҖ” CSV PROCESSING FLOW

When a CSV is uploaded to `POST /api/upload`:

```
1. Multer saves file to ./uploads/temp_TIMESTAMP.csv
2. Read file with utf-8-sig encoding
3. Split into lines (handle \r\n and \n)
4. Extract metadata from rows 2вҖ“3:
   - Row 3, col 1 (index 1): Meeting Topic вҶ’ parse subject and session type
   - Row 3, col 2 (index 2): Meeting ID вҶ’ use for duplicate check
   - Row 3, col 3 (index 3): Session date
5. CHECK DUPLICATE: query uploads table WHERE meeting_id = extracted_id
   - If found вҶ’ return error JSON with original upload date вҶ’ DO NOT SAVE
6. Find header row: scan lines for one containing 'User Name'
   - This is Row 10 (index 9)
7. Parse data rows from next line onward:
   - Skip if line is empty or starts with ','
   - Extract: name, email, submission_time, delivery_rating, content_rating, feedback_text
   - Detect cohort from email prefix
   - Classify feedback: isUseful вҶ’ sentiment вҶ’ themes (call nlp.js)
8. Compute aggregated metrics:
   - avg_delivery = average of all delivery_rating values (exclude NULLs)
   - avg_content = average of all content_rating values (exclude NULLs)
   - avg_combined = (avg_delivery + avg_content) / 2
   - nps_delivery = (count of 5s - count of 1s and 2s) / total Г— 100
   - feedback_count = count of useful feedback items
9. INSERT INTO uploads (metadata + aggregated metrics)
10. INSERT INTO poll_responses (one row per student)
11. UPSERT INTO students (update if email exists)
12. Delete the temp CSV file
13. Return success JSON with summary stats
```

## рҹ”ў METRICS FORMULAS

```
Average Rating = SUM(ratings) / COUNT(non-null ratings)

NPS = ((Count of 5-star ratings) - (Count of 1-star + 2-star ratings)) / Total Г— 100

Participation = Total Responses (just a count)

Feedback Rate = (Count of non-empty feedback / Total Responses) Г— 100

Session Satisfaction = (Avg Delivery + Avg Content) / 2

Trend Direction = Current avg - Previous session's avg for same subject
  If > 0.1 вҶ’ "вҶ‘ Improving"
  If < -0.1 вҶ’ "вҶ“ Declining"
  Else вҶ’ "вҶ’ Stable"
```

---

## рҹ”„ DUPLICATE DETECTION STRATEGY

Use `Meeting/Webinar ID` (Row 3, Column C of the CSV) as the unique key.
- Every Zoom session has a unique meeting ID
- Store it in the `uploads.meeting_id` column with a UNIQUE constraint
- On upload, query: `SELECT * FROM uploads WHERE meeting_id = ?`
- If match found: return error with original upload date
- If not found: proceed with processing

## вҡ пёҸ CRITICAL EDGE CASES TO HANDLE

1. **BOM Character**: Always use `utf-8-sig` or strip `\ufeff` from start of file
2. **CRLF vs LF**: Handle both `\r\n` and `\n` line endings
3. **Multiline fields**: Feedback text can contain commas and newlines inside quotes вҖ” use proper CSV parsing
4. **Empty ratings**: Store as NULL, never as 0 (to avoid skewing averages)
5. **Missing feedback**: `is_useful_feedback = 0` and `feedback_text = NULL` вҖ” do not crash
6. **Port binding on Replit**: Always bind to `0.0.0.0`, not `localhost`
7. **Database directory**: Create `./db/` folder if it doesn't exist before initializing SQLite
8. **Upload directory**: Create `./uploads/` folder if it doesn't exist
9. **React build on Replit**: Run `npm run build:client` before starting server
10. **CORS**: Enable CORS on all API routes for development
